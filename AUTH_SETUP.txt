Настройка за Вход / Регистрация / Табло (Firebase)

1. Firebase Console
   - Отвори https://console.firebase.google.com и избери проекта cleverhaus-petrov (или създай такъв).

2. Authentication
   - В менюто: Build → Authentication → Get started.
   - В таб "Sign-in method" включи "Email/Password" (Enable).

3. Конфигурация за уеб приложението
   - Project settings (иконката зъб) → General → Your apps.
   - Ако нямаш уеб ап, натисни "</>" и регистрирай нов. Копирай firebaseConfig обекта.
   - В проекта отвори firebase-config.js и замени стойностите с тези от конзолата:
     - apiKey
     - authDomain (напр. cleverhaus-petrov.firebaseapp.com)
     - projectId, storageBucket, messagingSenderId, appId
     - databaseURL трябва да е: https://cleverhaus-petrov-default-rtdb.europe-west1.firebasedatabase.app

4. Realtime Database – правила (сигурни)
   - Build → Realtime Database → Rules.
   - Копирай съдържанието от firebase-database-rules.json в проекта (или от тук):
     {
       "rules": {
         "users": {
           "$uid": {
             ".read": "auth != null && auth.uid === $uid",
             ".write": "auth != null && auth.uid === $uid"
           }
         }
       }
     }
   - Така всеки логнат потребител има достъп само до своите данни под users/<неговият-uid>.
   - Публикувай правилата (Publish).

5. Съвместимост с Arduino
   - Уеб приложението използва път /users/<uid>/ (Firebase Auth UID на потребителя).
   - Arduino трябва да записва под същия път: /users/<uid>/devices/, /users/<uid>/history/ и т.н.
   - UID-то виждаш във Firebase Console → Authentication → Users (колона User UID) или временно в конзолата на браузъра след вход: firebase.auth().currentUser.uid
   - Настрой скицата на Arduino да използва този UID вместо safeEmail в пътищата.

Страници:
- login.html   – Вход (имейл + парола)
- register.html – Регистрация (създаване на профил)
- reset-password.html – Смяна на парола от линка в имейла (нова парола + потвърждение)
- dashboard.html – Табло след вход: режим У дома / Навън и списък сензори. Бутон „Изход“.

От главната страница (index.html) връзката „Вход“ в навигацията води до login.html.

6. Имейл „Забравена парола“ (password reset) – английски текст + две полета за парола
   - Писмото изпраща Firebase. В login.html преди изпращане се задава languageCode = „en“, за да е текстът в имейла на английски.
   - Линкът в имейла трябва да отваря reset-password.html с параметри mode=resetPassword и oobCode=… Затова в Firebase Console:
     Authentication → Templates → Password reset → редактиране (молив)
     → „Customize action URL“ / поле за Action URL: пълният адрес към reset-password.html, напр.:
       https://aurahomesystems.eu/reset-password.html
     (за локален тест временно може http://localhost:3000/reset-password.html – шаблонът позволява един URL; за production върнете production адреса.)
   - В същия шаблон изберете език English за съдържанието на имейла, ако има избор.
   - В кода sendPasswordResetEmail подава същия handler URL (reset-password.html) чрез actionCodeSettings.url – доменът трябва да е в Authorized domains.
   - Authentication → Settings → Authorized domains: localhost, вашият production домейн и др.
   - Ако Action URL остане по подразбиране, линкът отваря страницата на Firebase с едно поле за парола – нашата reset-password.html няма да се ползва.
   - Проверете „Спам“; при липса на писмо – тест от Authentication → Users → Reset password.
   - Защита „Email enumeration“: при грешен имейл понякога няма писмо, но няма и ясна грешка – ползвайте точния имейл от Users.

   Линкът в спам не се отваря с клик, но работи при копиране в браузър
   - Това не е грешка в сайта: в папка „Спам“/„Нежелана поща“ много клиенти (Gmail, Outlook, Yahoo) изключват активните линкове. Кликът „не прави нищо“; копирането на пълния URL в Chrome/Firefox работи.
   - Решения: маркирайте подателя като „Не е спам“ / преместете писмото във входящи и опитайте отново; или винаги копирайте линка.
   - Action URL в шаблона трябва да е пълен адрес с https:// и домейна ви (не относителен път), напр. https://aurahomesystems.eu/reset-password.html
   - В редактора на същия шаблон (Password reset) добавете под бутона абзац с видим пълен линк за копиране, напр. текст: „If the button does not work, copy this link:“ и на следния ред само %LINK% (Firebase замества с реалния URL). Така при блокиран клик в спам потребителят копира адреса от текста.
